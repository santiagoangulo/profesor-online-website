# seo Specification

## Purpose

Document the SEO (Search Engine Optimization) implementation for the website. This includes meta tags, structured data, and accessibility features that contribute to search visibility and social sharing.

## Requirements

### Requirement: Page titles

Each page SHALL have a unique, descriptive title element.

#### Scenario: Unique page titles

- GIVEN any page on the site
- THEN the `<title>` element SHALL be populated
- AND the title SHALL be unique across all pages
- AND the title SHALL accurately describe the page content

#### Scenario: Title structure

- GIVEN a visitor views a page title
- THEN the title SHALL follow the format: "Page Specific Title | Profesor Online"
- OR the title SHALL be specific enough to stand alone without the brand name

### Requirement: Meta descriptions

Each page SHALL have a relevant meta description for search results.

#### Scenario: Meta description presence

- GIVEN any page on the site
- THEN a `<meta name="description">` tag SHALL be present
- AND the description SHALL be between 120-160 characters
- AND the description SHALL accurately summarize the page content

### Requirement: Viewport meta tag

The viewport meta tag SHALL be present for mobile responsiveness.

#### Scenario: Viewport configuration

- GIVEN any page on the site
- THEN `<meta name="viewport" content="width=device-width, initial-scale=1.0">` SHALL be present
- AND the page SHALL render correctly on mobile devices

### Requirement: Canonical URLs

Pages with potential duplicate URLs SHALL have canonical link tags.

#### Scenario: Canonical tag presence

- GIVEN a page has a canonical URL defined
- THEN a `<link rel="canonical">` tag SHALL be present in the `<head>`
- AND the canonical URL SHALL point to the preferred version of the page

### Requirement: Open Graph meta tags

Pages SHALL have Open Graph tags for social media sharing.

#### Scenario: OG title and description

- GIVEN a page is shared on social media (Facebook, LinkedIn)
- THEN `<meta property="og:title">` SHALL contain the page title
- AND `<meta property="og:description">` SHALL contain the page description

### Requirement: HTML lang attribute

The HTML document SHALL have a lang attribute reflecting the current language.

#### Scenario: Lang attribute on Spanish pages

- GIVEN the site is displaying Spanish content
- THEN `<html lang="es">` SHALL be present

#### Scenario: Lang attribute on English pages

- GIVEN the site is displaying English content
- THEN `<html lang="en">` SHALL be present

### Requirement: Robots meta tag

The site SHALL allow search engine crawling.

#### Scenario: Robots configuration

- GIVEN a search engine bot visits any page
- THEN no `<meta name="robots">` tag SHALL block indexing
- AND the `robots.txt` file SHALL allow crawling of main pages

#### Scenario: Netlify configuration

- GIVEN the site is deployed on Netlify
- THEN robots.txt configuration SHALL be handled by Netlify defaults
- AND the `netlify.toml` file SHALL be used for deployment settings

### Requirement: Sitemap

A sitemap SHALL be available for search engines.

#### Scenario: Sitemap presence

- GIVEN a search engine bot visits `/sitemap.xml`
- THEN an XML sitemap SHALL be returned
- AND the sitemap SHALL list all main site pages

## Technical Implementation

- Title and description are defined in each page's front matter
- Base template (`_includes/layouts/base.njk`) renders meta tags
- Sitemap is auto-generated by 11ty from content

## Out of Scope

- Structured data (JSON-LD)
- Twitter Card meta tags
- hreflang implementation
